Маршрутизация задач для Claude Code с оглядкой на стоимость. Четыре субагента, каждый закреплён за своим тиром модели, скилл выбирает нужный под каждую задачу, и хуки форсируют правила маршрутизации и границ контекста механически, а не полагаются на то, что модель сама вспомнит.

## Что внутри

**Четыре субагента** (`agents/`), каждый закреплён за своим тиром модели:

- `cheap`: полностью специфицированная механика: переименования, чистки, бойлерплейт, правки доков, переводы. Хорошо работает как параллельный fan-out по списку мелких кусков
- `dev`: дефолтный исполнитель самодостаточной реализации: фич, скоуп-рефакторинга, билд- и ран-проверок. Та же модель, что и основная сессия; выигрыш в изоляции контекста и параллелизме, не в цене
- `hard`: известно-трудная работа: adversarial-ревью, гонки, миграции API и схем, staging-верификация синков, правки на 10+ связанных файлов
- `super`: предельный тир для подтверждённого провала `hard`, недробимой многочасовой автономной работы или максимальной цены ошибки. Спавн поднимает нативный диалог подтверждения

**Скилл маршрутизации** (`skills/model-routing/SKILL.md`) выбирает тир под каждую задачу, перечисляет, что вообще не делегируется (разрушающие git-операции, секреты, личные юридические и финансовые документы), и определяет, как передавать контекст, когда задача эскалирует на более дорогой тир. Он же управляет встроенными агентами `Explore` и `general-purpose` и параметром `model` у инструмента Agent, которые иначе молча наследуют модель сессии.

**`route-gate.sh`**, хук на каждый спавн `Agent`/`Task`. `Explore` без указанной модели автоматически идёт на haiku; `general-purpose` без модели блокируется, пока модель не названа; спавн `super` запрашивает подтверждение. Каждое решение логируется в `~/.claude/hooks/state/route-gate.log`. Выключатель: `touch ~/.claude/hooks/route-gate.disabled`.

**`context-check.sh`**, хук, который следит за размером контекста и, перейдя порог, просит Claude предложить: передать эстафету в новый чат, очистить текущий, или продолжить. Мягкий порог около 55% окна, жёсткий около 88%. Окно по умолчанию 1M, переопределяется `CONTEXT_CHECK_WINDOW`. Выключатель: `touch ~/.claude/hooks/context-check.disabled`.

**`CLAUDE.md.example`**, правило для твоего собственного `~/.claude/CLAUDE.md`, которое подключает предложение с границы контекста выше к поведению Claude.

**`/refresh-rules`** — перечитывает CLAUDE.md и скилл маршрутизации прямо в уже открытом чате, когда правишь их посреди сессии.

**`/usage-report`** — показывает отчёт по токенам и моделям за текущую сессию, посчитанный из транскрипта.

**`/routing-status`** — показывает таблицу того, что route-gate реально разрешил, переписал или заблокировал в этой сессии, плюс реальные суммы токенов по моделям.

## Как пользоваться командами

```
/routing-status
```

```
Маршрутизация:
  dev              x4   allow-tier
  cheap            x6   allow-tier
  Explore          x2   rewrite-haiku
  general-purpose  x1   deny-no-model

Токены:
  Sonnet 5    142 300  (61%)
  Haiku 4.5    58 900  (25%)
  Opus 4.8     32 100  (14%)
```

Рост `deny-no-model` значит, что агентов продолжают спавнить без названной модели. `rewrite-haiku` больше нуля значит, что автопереход Explore на haiku реально срабатывает.

```
/usage-report
```

То же самое, но проще: только итоговые токены по моделям за сессию.

```
/refresh-rules
```

Запусти это в чате, который открыт с тех пор, как ты последний раз правил CLAUDE.md или скилл маршрутизации, чтобы подхватить текущие правила без рестарта.

## Полезно знать

- Выбор модели, реальный рычаг. У субагентов нет отдельной ручки "effort"; глубина размышления по тирам симулируется промптом, а не форсируется параметром.
- В `SKILL.md` названы конкретные модели и цены для справки, проверяй актуальные цены отдельно, а порядок тиров (от дешёвого к дорогому) держи как несущую логику.
- Хуки — настоящие шелл-скрипты, которые запускаются на каждый ответ или спавн агента, в каждом проекте, сразу после установки. Прочитай их перед установкой.
- Описания тиров в `skills/model-routing/SKILL.md` отражают рабочий процесс, завязанный на разработку софта. Подстрой их под свой набор задач.
- Токены и статус маршрутизации доступны только по запросу, двумя командами выше — само собой ничего не печатается в конце ответа.

## Установка

### Как плагин (рекомендуется)

```
/plugin marketplace add reganomika/Bullpen
/plugin install bullpen@bullpen
```

Вместо `reganomika/Bullpen` точно так же работает путь к локальному клону. Разом регистрирует все четыре агента, все четыре скилла и оба хука. Перезапусти Claude Code (или `/reload-plugins`) один раз после установки; правки скиллов применяются на лету.

`CLAUDE.md.example` никогда не ставится автоматически: плагинная система не грузит файлы CLAUDE.md. Добавь его содержимое в свой `~/.claude/CLAUDE.md` вручную.

### Скопировать в свой конфиг (без плагинной системы)

```bash
git clone <this-repo-url>
cp <repo>/agents/*.md ~/.claude/agents/
cp -r <repo>/skills/model-routing <repo>/skills/usage-report <repo>/skills/refresh-rules <repo>/skills/routing-status ~/.claude/skills/
mkdir -p ~/.claude/hooks
cp <repo>/hooks/route-gate.sh <repo>/hooks/context-check.sh ~/.claude/hooks/
chmod +x ~/.claude/hooks/route-gate.sh ~/.claude/hooks/context-check.sh
```

Добавь в `~/.claude/settings.json` (слей со своим существующим файлом):

```json
{
  "hooks": {
    "Stop": [{ "hooks": [{ "type": "command", "command": "~/.claude/hooks/context-check.sh" }] }],
    "PreToolUse": [{ "matcher": "Agent|Task", "hooks": [{ "type": "command", "command": "~/.claude/hooks/route-gate.sh" }] }]
  }
}
```

Запусти новую сессию, чтобы подхватить агентов, скиллы и хуки. По желанию добавь `CLAUDE.md.example` в свой CLAUDE.md. Выключатель для любого хука: `touch ~/.claude/hooks/<имя>.disabled`, обратно `rm`.

### Попробовать без установки

```bash
claude --plugin-dir <path-to-this-repo>
```
