Маршрутизация задач для Claude Code с оглядкой на стоимость: четыре субагента, каждый закреплён за своим тиром модели, скилл решает, кого из них звать, и два хука, которые форсируют правила маршрутизации и границ контекста механически, а не оставляют это на прозу. Токены и статус маршрутизации по запросу, двумя слэш-командами, а не в каждом ответе.

## Какую проблему это решает

Claude Code легко гонять любую задачу на самой мощной модели, и так же легко забыть, что делаешь именно это. В долгих агентных сессиях токены, потраченные на перечитывание накопленного контекста, обычно на порядок превышают токены на сам вывод. Выбор модели важен. Сколько истории тащишь за собой, обычно важнее.

Bullpen отправляет работу на самую дешёвую модель, которая с ней справится: поиск, разведку и механику на дешёвый тир (форсирует гейт на пути спавна агента, не совет в промпте), тогда как делегирование той же модели покупает изоляцию контекста, а не скидку. Две команды по запросу показывают, что реально запускалось, модель за моделью.

## Что внутри

**Четыре субагента** (`agents/`), каждый закреплён за своим тиром модели:

- `cheap`: полностью специфицированная механика: переименования, чистки, бойлерплейт, правки доков, переводы. Сильнее всего работает как параллельный fan-out по списку мелких кусков
- `dev`: дефолтный исполнитель самодостаточной реализации, фич, скоуп-рефакторинга, билд- и ран-проверок. Та же модель, что и основная сессия, поэтому даёт изоляцию контекста и параллелизм, а не экономию
- `hard`: известно-трудная работа, вход проактивный: adversarial-ревью, гонки, миграции API и схем, staging-верификация синков, 10+ связанных файлов
- `super`: предельный тир для подтверждённого провала `hard`, недробимой многочасовой автономной работы или максимальной цены ошибки. Спавн поднимает нативный диалог подтверждения

**Скилл маршрутизации** (`skills/model-routing/SKILL.md`) решает, какой тир звать, что вообще не делегировать (разрушающие git-операции, секреты, личные юридические и финансовые документы), и как передавать контекст между эскалирующими тирами. Он же управляет встроенными агентами `Explore` и `general-purpose` и параметром `model` у инструмента Agent, которые иначе молча наследуют модель сессии.

**`route-gate.sh`**, хук на `PreToolUse` для вызовов `Agent`/`Task`. `Explore` без `model` переписывается на haiku; `general-purpose` без `model` отклоняется, пока модель не названа; `super` поднимает диалог подтверждения. Каждое решение логируется в `~/.claude/hooks/state/route-gate.log`; тишина в логе при явном спавне агентов значит, что гейт мёртв (fail-open по умолчанию). Выключатель: `touch ~/.claude/hooks/route-gate.disabled`.

**`context-check.sh`**, хук на `Stop`, читает размер контекста из транскрипта и, перейдя порог, блокирует ответ, пока Claude не поднимет чекпоинт границы задачи. Мягкий порог (~55% окна) это текстовая подсказка, которую Claude может отклонить ради непрерывной работы; жёсткий (~88%, до автосжатия харнесса) форсирует настоящий вызов `AskUserQuestion` и повторяется, пока чекпоинт не появится в транскрипте. Окно по умолчанию 1M, переопределяется `CONTEXT_CHECK_WINDOW`. Выключатель: `touch ~/.claude/hooks/context-check.disabled`.

**`CLAUDE.md.example`**, опциональное правило, заставляющее Claude на границе контекста предложить структурированный выбор (промпт-передачу, очистить и остаться, или продолжить) вместо того, чтобы молча тащить раздутый разговор дальше. `context-check.sh` даёт числовой триггер, Claude по-прежнему сам судит, закрыта ли задача.

**`/refresh-rules`** перечитывает CLAUDE.md и скилл маршрутизации прямо в уже открытом чате, когда правишь их посреди сессии и не хочешь рестарта.

**`/usage-report`** показывает отчёт по токенам за сессию, посчитанный из транскрипта, по запросу.

**`/routing-status`** проверяет, что маршрутизация реально работает: читает `route-gate.log`, что прогонялось через гейт в этой сессии, и транскрипт для реальных сумм токенов по моделям, независимо от того, что чат сам о себе говорит.

## Команды

`/refresh-rules`, `/usage-report` и `/routing-status` запускаются вручную, ничего здесь не срабатывает само. Раньше проект форсировал отчёт по токенам в конце каждого ответа через `Stop`-хук, убрано 21.07.2026 после бага с дублирующейся строкой из-за привычки модели, который чисто текстовый фикс не держал стабильно. Теперь цифры показываются только когда просят.

```
/routing-status
```

```
Маршрутизация:
  dev              x4   allow-tier
  cheap            x6   allow-tier
  Explore          x2   rewrite-haiku
  general-purpose  x1   deny-no-model

Токены:
  Sonnet 5    142 300  (61%)
  Haiku 4.5    58 900  (25%)
  Opus 4.8     32 100  (14%)
```

Рост `deny-no-model` значит, что агентов продолжают спавнить без названной модели. `rewrite-haiku` больше нуля значит, что автопереход Explore на haiku реально срабатывает.

## Честные ограничения

- **У субагентов нет настоящей ручки "effort" на задачу.** Выбор модели, реальный рычаг; глубина размышления по тирам симулируется инструкциями в промпте, а не форсируется параметром.
- **Названия моделей и цены устареют.** Держись относительного порядка в `SKILL.md`, это несущая логика; проверяй актуальные цены отдельно.
- **Это настоящие шелл-скрипты, которые запускаются на каждый ответ или спавн агента, в каждом проекте, сразу после установки.** Прочитай их перед установкой.
- **Это отражает один рабочий процесс, не универсальный.** Подстрой описания тиров в `SKILL.md` под свой набор задач.
- **`context-check.sh` форсирует настоящий вызов инструмента только на жёстком пороге; мягкий это текстовая подсказка,** которую Claude может отклонить, и он проверяет размер только на границах хода, так что один ход может проскочить прямо к автосжатию.
- **Форсированный отчёт после каждого ответа пробовали и отказались от него** из-за бага с дублирующейся строкой, который текстовый контрпример не держал стабильно. У команд по запросу этой проблемы нет, но спросить нужно самому, само оно больше не появится.

## Установка

### Как плагин (рекомендуется)

```
/plugin marketplace add reganomika/Bullpen
/plugin install bullpen@bullpen
```

Вместо `reganomika/Bullpen` точно так же работает путь к локальному клону. Разом регистрирует все четыре агента, все четыре скилла и оба хука (`context-check.sh` на `Stop`, `route-gate.sh` на `PreToolUse`). Перезапусти Claude Code (или `/reload-plugins`) один раз после установки; правки скиллов применяются на лету.

`CLAUDE.md.example` никогда не ставится автоматически: плагинная система не грузит файлы CLAUDE.md. Добавь его содержимое в свой `~/.claude/CLAUDE.md` вручную.

### Скопировать в свой конфиг (без плагинной системы)

```bash
git clone <this-repo-url>
cp <repo>/agents/*.md ~/.claude/agents/
cp -r <repo>/skills/model-routing <repo>/skills/usage-report <repo>/skills/refresh-rules <repo>/skills/routing-status ~/.claude/skills/
mkdir -p ~/.claude/hooks
cp <repo>/hooks/route-gate.sh <repo>/hooks/context-check.sh ~/.claude/hooks/
chmod +x ~/.claude/hooks/route-gate.sh ~/.claude/hooks/context-check.sh
```

Добавь в `~/.claude/settings.json` (слей со своим существующим файлом):

```json
{
  "hooks": {
    "Stop": [{ "hooks": [{ "type": "command", "command": "~/.claude/hooks/context-check.sh" }] }],
    "PreToolUse": [{ "matcher": "Agent|Task", "hooks": [{ "type": "command", "command": "~/.claude/hooks/route-gate.sh" }] }]
  }
}
```

Запусти новую сессию, чтобы подхватить агентов, скиллы и хуки. По желанию добавь `CLAUDE.md.example` в свой CLAUDE.md. Выключатель для любого хука: `touch ~/.claude/hooks/<имя>.disabled`, обратно `rm`.

### Попробовать без установки

```bash
claude --plugin-dir <path-to-this-repo>
```
